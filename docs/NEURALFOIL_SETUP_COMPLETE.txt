  • Maximum L/D (overall best performance)
  • Regime-specific L/D (cruise, climb, high-α, negative-α)
  • CLmax and stall angle (maximum lift capability)
  • Minimum CD (lowest drag achieved)
  • Design-point performance at estimated Reynolds number
  • L/D coefficient of variation (consistency metric)
  • Analysis confidence (quality score)

🎓 INTERPRETING RESULTS
════════════════════════════════════════════════════════════════════════

L/D RATIO BENCHMARKS:
  Excellent : L/D > 30   (very efficient airfoil)
  Very Good : L/D 20-30  (efficient)
  Good      : L/D 15-20  (moderate efficiency)
  Moderate  : L/D 10-15  (basic performance)
  Poor      : L/D < 10   (inefficient)

Note: Bird airfoils often have lower L/D than engineered airfoils due to:
  • Low Reynolds numbers (high viscous drag)
  • Thickness constraints (structural requirements)
  • Multi-point design (not optimized for single condition)

CLmax INTERPRETATION:
  High      : CLmax > 1.5   (high-lift, aggressive camber)
  Moderate  : CLmax 1.0-1.5 (balanced design)
  Low       : CLmax < 1.0   (low-drag, speed-oriented)

STALL ANGLE:
  Gentle    : Stall > 16°   (wide usable AoA range)
  Typical   : Stall 12-16°  (normal behavior)
  Sharp     : Stall < 12°   (limited envelope)

ANALYSIS CONFIDENCE:
  High      : > 0.7   (trustworthy prediction)
  Medium    : 0.5-0.7 (reasonable, use with caution)
  Low       : < 0.5   (uncertain, may be inaccurate)

❓ TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════

"ModuleNotFoundError: No module named 'neuralfoil'"
  → pip install neuralfoil

"Analysis taking too long"
  → Expected! Use subset analysis or reduce model size

"Memory error"
  → Reduce MODEL_SIZE from "xxlarge" to "large"

"Unusual L/D values"
  → Check analysis_confidence in results
  → Verify coordinates normalized correctly (script handles this)

"Process killed"
  → System may be out of memory
  → Run smaller subset or use MODEL_SIZE="medium"
